Preguntas del Laboratorio 1

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

La World Wide Web es todo el complejo de información digital que se puede encontrar en sitios interconectados por una red que cuenta con protocolos, convenciones y software. Cualquiera la puede explorar o añadirle elementos. Este complejo o red funciona o corre precisamente a través del internet, pues este no es más que el medio de comunicación entre 2 o más dispositivos para transferir información.

2. ¿Cuáles son las partes de un URL?

Una URL o "Uniform Resource Locator" es una forma de establecer la dirección web de un recurso. Está compuesta por el protocolo que usa (ej: http), el dominio (el nombre para identificar la(s) direcciones IP donde se encuentra el recurso), el "path" (la dirección del recurso DENTRO del servidor) y los parámetros (datos adicionales para identificar o filtrar el recurso).

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

-GET sirve para solicitar datos de un recurso de la web.
-HEAD funciona como GET, pero solo devuelve el encabezado (no el cuerpo o los datos del contenido), por lo que normalmente se usa para verificar que existe un recurso.
-POST sirve para mandar los datos a un servidor para crear un recurso.
-PUT sirve para actualizar o editar un recurso.
-PATCH sirve para aplicar modificaciones PARCIALES a un recurso.
-DELETE sirve para eliminar un recurso.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe de usar el método POST, ya que se están enviando los datos de un recurso al servidor del sitio.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, ya que a través de la URL se solicita al servidor los datos de la página que es desplegada por el navegador web.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, de acuerdo con los códigos de estatus de HTTP, el código de respuesta 200 implica que la solicitud que se le hizo al servidor se ejecutó o se cumplió con éxito y no hubo ningún error.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Normalmente sí, considerando que el error suele ser causado por errores en rutas, enlaces, nombres, etc. que causan una solicitud errónea. Estos errores los comete un desarrollador, pero él mismo puede corregirlos fácilmente.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Normalmente no, porque ese tipo de errores vienen del servidor y no de aquello generado por el desarrollador. Sin embargo, si el desarrollador es responsable o puede modificar el código del servidor, lo puede corregir.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo ya no se usa actualmente o ya no entra en los estándares actuales de la web y de sus protocolos. En la versión más actual de HTML (HTML5), están desaprobados elementos (usados en HTML4) que han sido mejorados o reemplazados por nuevos, como "name" (ahora se usa id),"rev" (ahora se usa rel), "usemap" (ahora se usa img), o atributos que ya no son necesarios como "version" o "target".

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 está adaptado a las tecnologías y protocolos o estándares actuales, siendo las diferencias principales con HTML4 la incorporación de APIs, la posibilidad de incluir videos y audios que se resproduzcan, las etiquetas semánticas de estructura moderna y mejores gráficos y herramientas de almacenamiento y soporte.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla en HTML5 se crea a partir de una etiqueta de "table", siendo este el componente principal que envuelve al contenido de la tabla. Sus componentes internos se pueden usar de formas o estructuras distintas, pero normalmente se usa la etiqueta "tr" para definir una fila dentro de la tabla, y dentro de esa etiqueta se puede usar la etiqueta "td" para definir los datos de una celda en esa fila (cada td en una siguiente columna). Además, se pueden añadir celdas de encabezado con "th", las cuales tienen un estilo centrado y en negritas.

12. ¿Cuáles son los principales controles de una forma HTML5?

Los principales controles incluyen "label" para leyendas, "input" para diferentes tipos de input (texto, archivos, input de sliders o pickers, etc.), "button" para botones, "select" para una lista de opciones, "output" para el resultado de un cálculo u operación, "fieldset" para unir o contener varios controles, entre otros.

14. ¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información va desde que el sistema es creado hasta que es eliminado, teniendo etapas como: 1. Adquisición o concepción, 2. Instalación o despliegue, 3. Funcionamiento y Mantenimiento, 4. Actualización o Cambio y 5. Eliminación o retirada, donde no todas las fases son completadas o se ejecutan de forma continua e irregular.

15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información se refiere específicamente al proceso para crear y desarrollar los sistemas, el cual es un proceso complejo. Se considera que sus fases son: 1.Análisis de requisitos y Planificación, 2.Diseño, 3.Desarrollo, 4.Pruebas, 5.Despliegue/Implementación y 6.Mantenimiento.
